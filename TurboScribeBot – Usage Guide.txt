üìò TurboScribeBot ‚Äì Usage Guide

This guide explains how to run **TurboScribeBot** inside Docker, configure it, and get results.

---

 1Ô∏è‚É£ Run the Container

Basic run command:

```bash
docker run \
  --env-file /hamada/TurboScribeBot/.env \
  -v /hamada/TurboScribeBot/outputs:/app/outputs \
 -v /hamada/TurboScribeBot/last_deleted.txt:/app/last_deleted.txt \
  turboscribe-bot-2 \
  [ARGS...]
```

`--env-file /hamada/TurboScribeBot/.env` ‚Üí (path to the .env file) provides credentials (EMAIL, PASSWORD).
`-v /hamada/TurboScribeBot/outputs:/app/outputs` ‚Üí mounts output folder to your current directory.
`[ARGS...]` ‚Üí script arguments (see below).

---

 2Ô∏è‚É£ Environment File (`.env`)

Create a `.env` file with your login info:

```env
EMAIL=your_email@example.com
PASSWORD=your_password
DAY=10
HOST_OUTPUTS_BASE=/hamada/TurboScribeBot/outputs
```

This avoids passing secrets on the command line.

---

 3Ô∏è‚É£ Arguments

Required

 `--id ID` ‚Üí numeric or unique ID for this process (used as output folder name).
 One of:
  `--link URL` ‚Üí YouTube/video/audio link.
  `--file PATH` ‚Üí Local file path (must be mounted into the container).

Sources

  Choose one workflow:

    Zoom / OneDrive workflow
      --source [zoom|onedrive] ‚Üí select the source.
      --passcode CODE ‚Üí Zoom recording passcode (if required).
      --with-transcription ‚Üí automatically transcribe after download (requires --language).

  Direct transcription workflow
    --link URL ‚Üí YouTube/video/audio link.
    --file PATH ‚Üí Local file path (must be mounted into the container).


Options

`--language LANG` ‚Üí language for transcription. (e.g., `en`, `ar`)
`--model [base|small|large-v2]` ‚Üí model size (default: `base`).

Features (flags)

`--speakers [N]` ‚Üí diarization (optional: number of speakers, default auto).
`--transcribe` ‚Üí transcribe the original audio language directly to English..
`--restore` ‚Üí enhance/restore audio quality.
`--timestamps ‚Üí add timestamps to result.
`--short_summary` ‚Üí generate short GPT summary.
`--detail_summary` ‚Üí generate detailed GPT summary.
`--translate LANG` ‚Üí translate transcript (e.g., `en`, `ar`).
`--download_audio` ‚Üí download original audio file.
